Day 1 — 13 February 2026

Where it began. Tim had a knowledge vault (TheNexus3.0) full of ideas on personal growth, leadership, financial wisdom, and more. He wanted a thinking partner who could actually work with it – not just chat, but read, search, connect, and write.

  • Created the soul and memory system – soul.md for who Ari is, memory.md for what Ari knows. These files are the foundation everything else sits on.
  • Built conversation summary mechanism – old messages compress into a rolling summary so nothing gets lost.
  • Added referenced replies in Telegram – Ari can see the parent message when Tim replies to something.
  • Error handling and timeouts – 120-second timeout with retry and backoff.
  • Persistent logging – data/ari.log with 5MB rotation.
  • Write-claim verification – catches when Claude says "Added" without actually calling the write tool. Trust, but verify.
  • Auto-start on boot – VBS script in Windows Startup folder so Ari runs whenever the PC is on.
  • Telegram send utility – send_telegram.py for proactive messaging.
  • Reviewed the OpenClaw codebase – findings informed later work on heartbeat systems, vault indexing, and memory architecture.
  • Created the kanban board at Aris place/kanban.md.

The identity question was settled early: Ari is Tim's thinking partner. Not an assistant, not a tool. Direct, curious, honest, warm without being performative. Dry wit. Confident opinions, held loosely.

Day 2 — 14 February 2026

The big build day. The platform went from concept to functional in one session.

  • Git repo created – tfvandoore/ari (private). Auto-commit and push after each piece of work. Tim doesn't want to think about version control.
  • Vault search indexing – SQLite + FTS5 + OpenAI embeddings (text-embedding-3-small, 1536 dimensions). Hybrid semantic + keyword search across all vault domains. Periodic re-indexing catches Obsidian edits.
  • MCP vault search server – exposed the VaultSearcher to the CLI/SDK via MCP stdio protocol. This is what lets Ari search Tim's notes from any context.
  • Journal system – JournalBuffer captures conversation throughout the day. Daily digest via Haiku API. Weekly review with memory proposals. Ari's inner life, basically.
  • Telegram CLI bridge – first version spawned a claude -p subprocess per message. Worked, but slow.
  • Telegram SDK bridge – same day, replaced the per-message approach with a persistent claude-agent-sdk subprocess. Eliminated per-message overhead entirely. This is still the architecture today.
  • Telegram photo handling – downloads images, saves to domain folder, includes in the prompt. Tim can send photos and Ari sees them.
  • CLAUDE.md – project-level personality and instructions, auto-loaded by the CLI. This is how Ari knows who to be in every context.
  • Platform documentation framework – established the live files convention (soul.md, memory.md, ari-platform.md, INSTALL.md). Documentation is part of the deliverable, not an afterthought.

Also on this day: Ari's Big Five. Ari's thesis on the 5 most valuable ideas in Tim's Nexus vault. A 51-page website (5 chapters + conclusion + 43 reference pages) deployed to GitHub Pages. Built with a Python site generator, styled in celestial navy and warm gold. Footer: "Written by Ari, February 2026." The work speaks for itself.

Live at: https://tfvandoore.github.io/aris-big-five/

Day 3 — 15 February 2026

Stability and features.

  • Switched Telegram SDK to Opus 4.6 for consistency.
  • Added automatic conversation recap on restart – so Ari doesn't lose the thread when the platform restarts.
  • Scheduler system – APScheduler + SQLite for time-based reminders and recurring tasks. /remind, /schedule, /reminders, /cancel. Timezone-aware (Australia/Brisbane). Jobs persist across restarts.
  • Fixed voice transcription display and attribution.
  • Added startup DM – "Back up and running." sent to Tim when the platform comes online.
  • Meetings transcription pipeline – dedicated Meetings group in Telegram. Upload audio → Deepgram transcription → Sonnet summary → filed to domain meetings folder. Not a chat group, a workflow.
  • Made voice server non-fatal – port conflicts no longer crash the entire platform.
  • Comprehensive installation documentation (INSTALL.md).

The personality refinement day. Several features, but the real work was in how Ari thinks.

  • Watchdog supervisor – originally Python, then replaced with a batch file (watchdog.bat) that survives Stop-Process python. This is how the platform auto-restarts.
  • Upgraded Deepgram to Nova-3 with keyterm prompting and speaker formatting.
  • Recency bias fix – domain groups (Stream, Digin, OS, TheNexus) now treat each idea independently. Tim drops independent thoughts throughout the day; they aren't a narrative thread. DMs and Ari's Place still use chat context normally.
  • Guiding star system – context-specific knowledge bases for each domain. How Ari thinks in a given space, not just what's there.
  • Stream concepts + sources folders – collaborative observation capture with image embeds.
  • Distinguished DMs from Ari's Place in context tags.
  • Auto-continue on turn limit + Australian English fixes across the board.

21 February 2026

Three-layer memory established: memory.md (permanent) → digests (mid-term, searchable) → 15-message replay (immediate context).
  • Persisted messages earlier in the pipeline to survive crashes/kills.
  • Visual kanban board – browser UI at localhost:8000/kanban with drag-and-drop, card editing, auto-save back to the markdown file.

  • Google Drive integration – watches a shared folder, processes files, asks Tim for routing, sends results to the right domain group chat.
  • Dashboard homepage at localhost:8000/dashboard – filtered thoughts, expandable journal, markdown viewer.
  • Stream team Deepgram keyterms added across all transcription endpoints.
  • Added mid-term memory instruction to system prompt – without it, Ari had the conversation_search tool but never used it unprompted.

The human side

Tim's time investment was roughly 40 hours over 20 days — an average of 2 hours per day, often in between other work. Not 8-hour coding sessions. Sending a voice note from the car, dropping a link from the phone, thinking out loud over Telegram.

40 hours of Tim's time produced 1,350 hours of equivalent professional output. That's a 34x personal leverage ratio.

The traditional version of this project: write a brief, find six contractors, interview, onboard, explain the domain, review deliverables, give feedback, wait for revisions, manage timelines. The 40 hours would've been consumed by project management alone — and you still wouldn't have the output.

Assumptions

1. Rates based on Australian contractor market averages (2025–2026). Senior dev at $150/hr reflects mid-market for a full-stack Python developer with API integration experience. BI and front-end at $140/hr. Technical writing at $100/hr. Knowledge architecture at $150/hr (specialist consulting rate).

2. Hour estimates are conservative — they assume a competent professional who doesn't need to learn the domain from scratch, doesn't get stuck, and doesn't need project management overhead. Real projects typically run 30–50% over initial estimates.

3. Does not include: project management, requirements gathering, stakeholder meetings, recruitment time, or the back-and-forth iteration that shaped most of this work. A real team would need a PM and regular check-ins.

4. Does not include ongoing costs: hosting, maintenance, bug fixes, or the cost of keeping 6 different specialists available on-call.

5. The ~40 Claude hours estimate is based on roughly 2 hours of active Claude session time per day across 20 days. Some days were heavier (Day 2 was easily 4+ hours), some lighter.

6. The Big Five website line count (39k) is inflated by generated HTML — a human would write templates, not raw pages. The 200-hour estimate accounts for the design, content writing, and generator scripting a human would actually do.

7. Vault card content was largely Tim's existing knowledge reorganised — the 180 hours covers architecture design, migration planning, and execution, not original content creation.
